# 🧑‍💻 새 사용자 온보딩 가이드

> 카카오톡 채널 챗봇으로 가입 → 관리자 승인 → PC 설치, 3단계.
> 별도 토큰 발급 도구 실행 없이 브라우저 + 챗봇만으로 진행.

---

## 📋 사전 준비

- 본인 카카오 계정 (개발자센터 가입 가능)
- 윈도우 PC (PowerShell 5.1+)
- NAS 운영자(=관리자)와 사전 합의 — 카카오 채널 친구 추가 + 본인 사용자명 (예: `alice`) 합의

---

## 1️⃣ 본인 카카오 개발자 앱 만들기 (1회)

각 사용자는 **자기 카카오 계정**으로 카카오 개발자 앱을 만들어야 합니다.

> 카카오톡 '나에게 보내기' 메시지는 **OAuth 토큰을 동의한 카카오 계정**에게만 도착하기 때문에, 본인 카톡으로 받으려면 본인 계정으로 동의 절차를 거쳐야 합니다.

### 절차

1. <https://developers.kakao.com> 본인 카카오 계정으로 로그인
2. **내 애플리케이션 → 애플리케이션 추가하기**
   - 앱 이름: 자유 (예: "내 Claude 알림")
   - 사업자명: 개인이면 본인 이름
3. **앱 설정 → 팀 관리 → 팀원 추가** → **본인 카카오 계정 이메일** 추가
   (talk_message 권한이 검수 미통과 앱에서는 팀원만 사용 가능)
4. **앱 키 → REST API 키** 복사 → 메모장에 임시 저장
5. **제품 설정 → 카카오 로그인 → 활성화 설정** → **ON**
6. **제품 설정 → 카카오 로그인 → Redirect URI** 에 다음 추가:
   ```
   https://dhub-ds.synology.me:8003/oauth/callback
   ```
   > ⚠️ NAS 운영 호스트가 다르면 관리자에게 정확한 redirect URI 를 받으세요. 위 URL은 dhub NAS 기준.
7. **제품 설정 → 카카오 로그인 → 동의항목** → **카카오톡 메시지 전송 (talk_message)** → **사용 설정** ON (필수 동의 또는 선택 동의)
8. (선택) **앱 설정 → 보안 → Client Secret** 발급 + 활성화 ON → 메모장에 저장

## 2️⃣ 카카오 채널 챗봇으로 가입 신청

NAS 운영자가 운영하는 카카오 채널을 친구 추가한 뒤:

```
/가입 alice
```

→ 봇이 **등록 URL** 을 회신:
```
🆕 가입 시작 — 'alice'
━━━━━━━━━━━━━━
아래 URL 을 브라우저로 여세요:

https://dhub-ds.synology.me:8003/register?req=...

카카오 앱 정보 입력 → 카카오 로그인 → 동의 →
관리자 승인 대기.
```

> 사용자명 규칙: 영문/숫자/`_`/`-` 1~64자. 다른 사용자와 중복 불가. 한 번 결정하면 변경 불가능.

---

## 3️⃣ 브라우저에서 카카오 앱 정보 입력

봇이 알려준 URL 을 브라우저로 열기. 폼에:

| 항목 | 값 |
|------|------|
| REST API key | 1단계의 4번에서 복사한 값 |
| Client Secret | 8번에서 발급했으면 입력, 안 했으면 비워두기 |

**[다음 → 카카오 로그인]** 클릭 → 카카오 로그인 페이지 자동 이동 → 로그인 + 권한 동의 → **OAuth 완료** 페이지.

> ⚠️ 이 폼에서 입력한 값은 카카오톡으로 전송되지 않으며, NAS 서버에만 안전 저장됩니다 (HTTPS).

OAuth 가 완료되면 자동으로 **관리자에게 승인 요청 카톡이 전송**됩니다.

---

## 4️⃣ 관리자 승인 대기 → 결과 확인

다시 카카오 채널 챗봇에서:

```
/가입상태 alice
```

가능한 응답:

- **⏳ 관리자 승인 대기** — 잠시 기다린 후 다시 입력
- **✅ 승인 완료** — API 키 + 페어링 코드 + PC 설치 명령 모두 회신
- **❌ 거부됨** — 관리자에게 사유 문의
- **⌛ 만료됨** — `/가입` 으로 다시 시작 (7일 안에 승인 안 되면 만료)

승인 완료 응답 예:
```
✅ 승인 완료
승인 시각: 2026-04-29T15:30:00

━ API 키 (PC 클라이언트용) ━
gwQmvSjtsAV9T7ZOf0wlgXTzvfNnJfio_i5thNQSwls

━ 페어링 코드 (다른 디바이스용, 10분) ━
ABC123

PC 설치:
git clone https://github.com/parkbohyun/claude-kakao-notify
cd claude-kakao-notify
powershell -ExecutionPolicy Bypass -File .\install.ps1

host: dhub-ds.synology.me / port: 8003
```

> 💡 처음 가입한 카카오 챗봇 user 는 **자동 페어링** 됩니다. 별도 `/연동` 입력 불필요. 다른 디바이스/계정에서 토글하고 싶을 때 위 페어링 코드 사용.

## 5️⃣ PC 클라이언트 설치

```powershell
git clone https://github.com/parkbohyun/claude-kakao-notify.git
cd claude-kakao-notify
powershell -ExecutionPolicy Bypass -File .\install.ps1
```

설치 중 입력:

| 항목 | 입력 값 |
|------|---------|
| NAS host or IP | `dhub-ds.synology.me` |
| NAS port | `8003` |
| API key | 4단계에서 받은 키 |

설치 완료 후 **Claude Code를 새로 띄우면** 첫 알림이 본인 카카오톡 '나에게 보내기'에 도착합니다 🎉

## ✅ 동작 확인

Claude Code 채팅에서:

```
/remote-control
```
URL이 표시되면:
```
/rcd
```
→ 카톡으로 "🔗 [Claude Code] 원격 접속 URL" 메시지가 도착해야 정상.

## 🔕 알림 ON/OFF 토글

회의·취침 중 등 잠시 알림을 끄고 싶을 때 카카오 채널에서:

```
/알림 끄기      → OFF (Claude Code 알림이 도착하지 않음)
/알림 켜기      → ON
/알림 상태      → 현재 상태 확인
```

> 가입 승인 시 자동 페어링 됐으므로 별도 `/연동` 절차 없이 바로 토글 가능.

OFF 상태에서도 PC 클라이언트는 정상 동작 (NAS 까지 요청은 가도 카톡 발송이 스킵됨).

### 다른 디바이스 추가 페어링 (선택)

회사 카톡과 개인 카톡 양쪽에서 토글하고 싶다면:

```
/코드요청 alice
```
→ "관리자 승인 대기" → 관리자가 승인 → `/코드확인 alice` → 새 페어링 코드 → 그 디바이스의 카카오 채널에서 `/연동 <코드>`.

## 🔧 문제 발생 시

| 증상 | 점검 |
|------|------|
| 가입 신청 후 1시간 지남 | 만료됨. `/가입 <name>` 으로 재시도 |
| 카카오 로그인 시 KOE006 | 1단계 3번 (팀 관리 본인 추가) + 5번(활성화 ON) 재확인 |
| OAuth 콜백에서 오류 | Redirect URI 가 정확히 `https://dhub-ds.synology.me:8003/oauth/callback` 인지 (오타·트레일링 슬래시 주의) |
| 설치 후 카톡 안 옴 | `~/.claude/hook-debug/SessionStart_*.json` 파일 존재 여부 → 있으면 PC 정상, 관리자에게 NAS 로그 확인 요청 |
| `Invalid API key` (401) | API 키와 `~/.claude/notify-api.env` 의 키 정확히 일치 (앞뒤 공백 포함) |
| `Connection refused` | host/port 확인. `curl https://dhub-ds.synology.me:8003/` 도달성 확인 |
| 카톡이 다른 사람에게 도착 | 1단계의 카카오 로그인을 본인 계정으로 했는지 재확인 |

## 🙋 자주 묻는 질문

**Q. 가입 시 입력한 client_id/secret이 카카오톡 채팅에 노출되나요?**
A. 아니오. 챗봇에는 사용자명만 입력하고, 비밀 값은 **HTTPS 웹폼**으로만 전송되어 NAS 서버에 저장됩니다.

**Q. 토큰을 갱신해야 하나요?**
A. access_token이 만료되면 NAS 컨테이너가 refresh_token으로 자동 갱신합니다. refresh_token 자체가 만료되는 경우(약 60일 미사용)에만 OAuth 재진행이 필요.

**Q. 여러 PC에서 사용 가능?**
A. 네. 각 PC에서 `install.ps1`을 같은 host/port/key로 실행하면 됩니다.

**Q. 설치를 제거하려면?**
A. `powershell -ExecutionPolicy Bypass -File .\uninstall.ps1` 실행. 추가로 NAS 관리자에게 본인 테넌트 제거를 요청하세요.

**Q. 관리자가 승인 안 해주면?**
A. 7일 동안 승인 안 되면 자동 만료. 관리자에게 직접 연락해 진행 상황 확인.

**Q. 기존(레거시) 방식으로 가입한 사용자는?**
A. 그대로 동작합니다. 챗봇 가입은 신규 사용자용. 기존 사용자가 챗봇 토글 쓰려면 `/연동 <코드>` 흐름 사용 가능.
